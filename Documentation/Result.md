# Отчет о проведённом тестировании

В ходе выполнения тестирования мобильного приложения **"Мобильный хоспис"** были достигнуты следующие результаты:

## Тестовые сценарии

Разработано **61 тес-кейс**, охватывающих ключевые функциональные модули приложения:
- Авторизация пользователя
- Просмотр и фильтрация новостей
- Создание, редактирование и удаление новостей
- Работа с панелью управления (Control Panel)
- Валидация полей формы
- Проверка состояний элементов (активен/неактивен, видим/скрыт)
- Обработка негативных сценариев

## Автоматизация тестов

Из общего количества **61 тест-кейсов — 58 покрыто автотестами**, что составляет **~97% автоматизации**.

Для автоматизации использовался фреймворк **Espresso** в связке с **JUnit4** + Faker. А также реализован паттерн **Page Object Model (POM)** для обеспечения читаемости, переиспользуемости и простоты поддержки тестового кода.

Все автотесты:
- Надёжны и устойчивы к ложным падениям
- Содержат явные ожидания и проверки видимости элементов
- Используют уникальные идентификаторы вместо текстовых матчеров
- Покрывают как UI-взаимодействия, так и бизнес-логику
- Используют Faker для генерации случайных данных
- Независимы друг от друга
- Реализованы в виде отдельных классовй



К проекту подключён инструмент **Allure**, который генерирует детализированные, наглядные HTML-отчёты с щагами выполнения тестов


## Результы сравнения времени проверки приложения по чек-листу руками и UI-тестами
 - Время затраченное на ручную проверку по чек листу - 2 часа
 - Время затраченное на автоматизированную проверку - 25 минут

## Заключение

Автоматизация тестирования показала высокую эффективность: **97% тест-кейсов покрыто автотестами**, что делает процесс проверки приложения быстрым, стабильным и воспроизводимым.

Ручное тестирование остаётся важным для поиска неочевидных багов и проверки UX, однако **автоматизация стала основным инструментом контроля качества**.

Выявленные дефекты требуют приоритетного исправления, особенно те, что связаны с корректностью отображения данных и работой RecyclerView.


## Рекомендации

1. **Продолжить расширение покрытия автотестами** — довести до 100%.
2. **Внедрить Allure + Jenkins/GitLab CI** для автоматического запуска тестов при каждом коммите.
3. **Добавить параметризацию тестов** через `@Parameterized` для проверки разных наборов данных.
4. **Регулярно рефакторить Page-классы** для поддержания чистоты и читаемости кода.
